HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • A declaration tells the browser what document type to expect when it is opened.
  • An attribute defines additional information about an element.
  • Metadata is typically not seen by users, but it contains important information about accessability, search engines, and performance.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside of an element.
  • A link element creates the connection to the external file.
  • An rel attribute specifies the relationship between the current document and the linked document resource.
  • An href attribute specifies the location (URL) of the external resource.

Git

  • git add [file]: Add a file as it looks now to your next commit (stage).
  • git status: Checks what branch we are currently on.
  • git checkout -b branch-name: Creates a new branch and switches to it.
  • git reset [file]: Unstage a file while maintaining the changes in the working directory.
  • git commit -m "[decriptive message]": Commit your staged content as a new commit snapshot.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • An array is a single variable that is used to hold a group of data.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
